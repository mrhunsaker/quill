## Pass 1: Developer Console (QDC) — COMPLETE

### Critical

1. BUG (crash risk): `ts_console._handle_invoke` runs host calls on the Node reader
   thread, and every `ConsoleHost` implementation in `main_frame_devtools.py` touches
   wx widgets directly (`self.editor.GetValue()`, `WriteText`, etc.). So
   `await quill.documentText()` reads a wx control from a background thread —
   violates the project threading invariant and is undefined behavior on wx.
   Fix: marshal invokes to the UI thread (wx.CallAfter + threading.Event wait).
   Safe because execute() runs on a daemon thread, so the UI thread is free.

2. BUG (crash risk): `restart_typescript_worker` menu item, when the console window
   was never opened, evaluates `self._dt_window()` on a daemon thread inside the
   CallAfter argument list — this CREATES a wx.Frame off the UI thread.
   Fix: resolve the window on the UI thread, or guard "no window yet".

3. BUG: "Open TypeScript Console" never switches the language selector — the
   ConsoleWindow Choice stays on Python, so typed TS executes as Python and
   syntax-errors. ConsoleWindow needs `set_language()`; both open_* must call it.

4. BUG (data loss for users): Up/Down arrows in the input field ALWAYS trigger
   history navigation, even mid-edit of a multi-line block. Arrowing between lines
   of your own input replaces it with a history entry. Docstring promises
   "when cursor at first/last line" but the code never checks. Especially bad for
   screen reader users who arrow to review what they typed.
   Fix: only trigger history when caret is on first (Up) / last (Down) line;
   otherwise event.Skip().

5. BUG: three console settings (`console_enabled`, `console_python_timeout`,
   `console_typescript_timeout`) are defined, schema-validated, and shown in the
   Settings dialog — but NOTHING reads them. Turning the console off does not
   disable it; timeouts are hardcoded at 30s. Dead settings shipped to users.

6. BUG (safety gap): Safe Mode does not disable the Developer Console. QDC runs
   arbitrary code — it is strictly more powerful than AI/Quillins, which Safe Mode
   does gate. `open_python_console`/`open_typescript_console` must check safe mode.

### High

7. BUG: `copy_diagnostic_summary` announces "Diagnostic summary copied to
   clipboard." even when `wx.TheClipboard.Open()` returned False and nothing was
   copied. SR user is told a falsehood. Same unchecked pattern in
   `console_window._copy_transcript` (which announces nothing at all — silent
   action for SR users).

8. BUG: node subprocess launched without `creationflags=CREATE_NO_WINDOW` on
   Windows — in the packaged (no-console) app a console window can flash.
   Also uses raw `subprocess.Popen` instead of `stability.safe_subprocess`
   (project invariant), and `stderr=PIPE` is never drained — if node writes
   enough to stderr the pipe fills and the worker deadlocks.
   Fix: safe_subprocess (or at least CREATE_NO_WINDOW + stderr=DEVNULL).

9. BUG: when a TS execution errors, the worker's `error` message includes an
   `output` field (console.log output captured before the throw) — `_dispatch`
   drops it. The user loses everything the script printed before failing.

10. A11Y: keyboard shortcuts (Esc, F1, Ctrl+L, Ctrl+Shift+C, Ctrl+S, history)
    are bound only to the input TextCtrl's EVT_KEY_DOWN. With focus on the
    transcript or any button, Esc does not close the window and F1 does not open
    help. Fix: bind EVT_CHAR_HOOK at frame level.

11. A11Y: Esc/close returns focus to `self._parent` (the main wx.Frame), not the
    editor control. PRD says "return focus to editor". Pass a focus-editor
    callback from DevToolsMixin instead.

12. BUG (dead/wrong code): `_on_enter` uses `wx.KeyboardState()` — a default-
    constructed state object that never reflects real key state, so the Shift
    check is always False. The handler is unreachable anyway (EVT_KEY_DOWN
    swallows Enter first). Delete it or use `wx.GetKeyState(wx.WXK_SHIFT)`.

### Medium

13. UX: first-run consent (`_dev_console_consent_shown`) is per-session, not
    persisted — users see the warning every launch. Persist in settings.
14. UX/dialog contract: consent + help use raw `wx.MessageBox` and the save
    dialog calls `ShowModal()` directly — bypasses `_show_modal_dialog`
    (z-order gate, focus contract, dialog inventory). The dialog inventory gate
    likely doesn't scan quill/devtools — blind spot worth closing.
15. SEC (privacy): `history.jsonl` stores every console line unredacted, forever
    (500 entries). A pasted API key lives in plaintext on disk. Run entries
    through `stability.redaction` before persisting, and document the file in
    the privacy docs. Also `_trim()` rewrites non-atomically (project invariant
    is atomic writes).
16. BUG: Python console history (`open_python_console`) loads ALL languages'
    history; TS path filters to typescript. Python should filter to python.
    Switching the language Choice also doesn't reload history.
17. A11Y: history navigation (Up/Down) replaces the input text silently — no
    announcement, and caret-end placement means SR users hear nothing. Announce
    the recalled line (or at least "history N of M").
18. UX: `q.help()` exists but nothing announces it on first open; new-user
    discoverability of the API is low. Consider announcing "Type q.help() for
    the API reference" on first open.
19. ENH: `drain_output()` in ts_console is dead code (output also arrives
    aggregated in done/error) — delete it or wire streaming output to the
    transcript for long-running scripts.
20. ENH: PythonConsole.push_line / _buffer / _needs_more are unused by the UI —
    either implement incremental multi-line entry (... continuation prompts)
    or remove.
21. UX: Python executes synchronously on the UI thread with no timeout — a
    `while True:` freezes QUILL with no recovery (wx_heartbeat will log, but
    user loses work). At minimum document this in the consent text; ideally
    run with a watchdog or at least warn in help text.

## Pass 2: GitHub integration — COMPLETE

What's good: token in Windows Credential Manager (never settings/disk), one-time
consent dialog before any network, helpful 401/403/404/409 error mapping, save-back
announces start and completion, dialog contract (apply_modal_ids/show_modal_dialog)
used by the four custom dialogs.

### Critical

22. A11Y (silent failures): GitHubRepositoryBrowserDialog communicates ALL state —
    "Loading...", "Error: ...", "N items in /" — only via a StaticText
    (`_set_status`). Screen readers do not announce label changes. An SR user who
    clicks Load on a bad repo name hears NOTHING, forever. Errors MUST be spoken.
    Fix: pass an announce callback into the dialog (MainFrame._announce) and call
    it for every status change, or at minimum for errors and load-complete.

23. BUG (wrong-repo commit): downloaded files all land at
    `github-temp/<basename>`. Opening README.md from repo A then repo B
    overwrites the temp file AND the `origins` dict entry (keyed by local path).
    Tab A's "Save to GitHub" now commits tab B's origin — wrong repository.
    Fix: unique per-(repo, ref, path-hash) subdirectory.

24. BUG (UI freeze): `open_github_repository`, `open_github_file_url`, and
    `manage_github_accounts` call `provider.get_identity()` — a synchronous
    GitHub network round-trip — on the UI thread, before any dialog shows.
    On a slow network the app freezes. The module docstring promises "All
    PyGithub calls run in daemon threads" — these don't.

25. BUG (data corruption): binary / non-UTF-8 files are decoded with a latin-1
    fallback, re-encoded as UTF-8, and written with newline="". Opening a
    non-text file mangles it; Save to GitHub then commits the mangled bytes
    over the original. Fix: detect binary (NUL check), refuse with a clear
    message; for latin-1 fallback files, block save-back or preserve encoding.

### High

26. BUG: repo-name TextCtrl is bound to EVT_TEXT_ENTER but lacks the
    wx.TE_PROCESS_ENTER style — Enter in the field never fires the Load handler
    on Windows; it hits the (initially disabled) default button instead.
    Keyboard-first users type a repo name, press Enter, nothing happens.

27. BUG (keyboard): the browser dialog's EVT_CHAR_HOOK grabs Backspace globally
    as "Go Up" whenever the path stack is non-empty — including while the user
    is EDITING the repository name field. Backspace stops deleting characters.
    Fix: only treat Backspace as Go Up when focus is on the list control.

28. A11Y (focus): after a directory loads, focus stays wherever it was and no
    list item is selected/focused. SR users get no cue that content arrived and
    must tab around. Fix: focus the list and select+focus item 0 after load —
    the selection event gives a free announcement.

29. UX edge: a repository with zero branches (empty repo) leaves the dialog
    permanently stuck — `_on_repo_loaded` with no refs sets `_current_ref=""`,
    `_load_directory` early-returns, and controls stay disabled from
    `_set_loading(True)`. Only Cancel works, with no explanation.

### Medium

30. Dialog contract: `open_github_file_url` and `github_save_back` call
    `dlg.ShowModal()` directly on wx.TextEntryDialog — bypasses
    `_show_modal_dialog` (z-order gate). CLAUDE.md: "never call ShowModal()
    directly."
31. SEC (process): PyGithub egress is invisible to network_egress_audit (it
    AST-scans for direct urllib/socket calls). Add a registry entry documenting
    the GitHub integration's egress surface so the audit stays honest.
32. SEC (privacy): downloaded GitHub files persist in app-data/github-temp
    indefinitely. Clean up on exit or document in PRIVACY.md.
33. BUG (minor): `_parse_github_blob_url` rejects URLs with query strings
    (?plain=1), line anchors (#L10) — both extremely common when copying from
    the GitHub UI — and doesn't unquote %20 in paths. Strip query/fragment,
    unquote path.
34. ENH: `list_refs` enumerates ALL branches and tags — on a repo with
    thousands (e.g. kubernetes) that's many paginated API calls before the
    dialog becomes usable, and burns anonymous rate limit (60/hr). Cap at ~100
    with "type to filter" or load default branch immediately and refs lazily.
35. ENH: `provider.close()` exists but no caller ever closes providers
    (created per-operation). Harmless today; tidy later.

## Pass 3: Autoupdate + release scripts — COMPLETE

What's good: the version-compare fix in vendor autoupdate (string compare ->
integer tuples) is correct and tested; build_update_zip places the bootstrapper
at ZIP root as the updater requires; manifests carry SHA-256 per file.

### Critical

36. SEC (supply chain): the release workflow runs `fetch_bootstrappers.py` with
    NO pinned baseline — `checksums.sha256` is UNTRACKED (not committed) and the
    script downloads `bootstrap.exe` from the HEAD of a third-party repo
    (AccessibleApps/app_updater), records whatever hash it got, and ships it in
    every release ZIP. The executable that performs updates on user machines is
    fetched unpinned at release time. If that repo is ever compromised, QUILL
    ships the attacker's binary.
    Fix (all three):
    a. Commit `checksums.sha256` with reviewed, known-good hashes.
    b. Pin BASE_URL to a specific commit SHA, not HEAD.
    c. In CI, run `fetch_bootstrappers.py` then `--verify-only` against the
       committed file and fail the release on mismatch.

### Medium

37. BUG (latent): `_version_tuple` in vendor autoupdate maps any non-numeric
    version ("1.2.3rc1", "2.0.0-beta") to (0,) — a server typo in the version
    field silently disables updates for everyone. Parse leading digits per
    segment instead of bailing to (0,).
38. ENH: `build_update_zip.py` has two near-identical base-version finders
    (`_find_latest_base_version` with packaging dep + `_simple`). Keep one.
39. Process: `quill/_vendor/autoupdate/bootstrappers/checksums.sha256` sits
    untracked in the working tree — decide: commit it (correct) or it silently
    does nothing. Stray root-level `leasey.html` is also untracked while
    `docs/leasey.html` was deleted — looks accidental; resolve before commit.

## Pass 4: Invariant sweeps — COMPLETE

What's good: all 131 dialog-contract tests pass; banned-pattern gate clean;
network egress audit clean; dialog focus-routing guard in place; menu mnemonics
present and no duplicate accelerators in the menu layer (accelerators live in the
keymap system, which has its own contract tests). The new .kqp keyboard-pack
import/export is clean: atomic writes, version check, delta-only storage.

40. Systemic: direct `threading.Thread(...)` appears in 15 UI files (SSH, AI
    panels, image, browse, GitHub, devtools...) despite the CLAUDE.md invariant
    that background work runs on QuillTaskManager. The new code copied the
    dominant local pattern. Decide once: either migrate to the task manager
    (gets you cancellation + shutdown draining) or amend the invariant. A CI
    gate (banned-pattern entry for `threading.Thread(` under quill/ui) would
    hold the line after migration.
41. Systemic: ~50 raw `wx.MessageBox` call sites bypass `_show_message_box`
    (z-order + announcement wrapper). Worst offenders are the newest files
    (devtools, console window). Same fix-and-gate approach as 40.
42. `import_keyboard_pack` merges and SAVES the keymap before the user sees
    what changed — no validation that binding values are parseable chords or
    that command ids exist (kqp_validator exists but is not in the import
    path). A bad .kqp silently breaks bindings. Validate + preview before save.

## Pass 5: Accessibility + UX (whole app) — COMPLETE

### Critical

43. A11Y (systemic — the single highest-impact finding in this review):
    status text set via `StaticText.SetLabel` is NEVER announced by screen
    readers, and at least four major dialogs use it as their only feedback
    channel:
    - ai_chat_dialog.py — 16 sites, 0 announces. "Fetching model list...",
      "Response received.", "Error — see details.", and every validation
      message ("Enter a prompt before sending.", "Select a model first.",
      "Save an API key first.") are all SILENT.
    - skill_library_dialog.py — 9 sites, 0 announces.
    - prompt_library_dialog.py — 5 sites, 0 announces.
    - github_dialogs.py — all status/errors silent (finding 22).
    - ssh_dialogs.py — 1 site, 0 announces.
    An SR user pressing Send in AI chat hears nothing until the response text
    appears — and hears nothing AT ALL if it errors. Validation errors that
    are never announced directly violate the project's own accessibility
    rules ("error messages must be announced").
    Fix: one shared helper (set label + announce in a single call), thread an
    announce callback into every dialog, and add a CI gate flagging
    `SetLabel` on status-named controls in files with zero announce calls.
    quillin_wizard.py (4 SetLabel / 6 announce) shows the pattern done right.

44. A11Y: setup wizard page changes are silent. On Next, focus jumps to the
    Next/Finish button and nothing announces the new page — an SR user on the
    nine-page first-run wizard hears "Next button" nine times and must
    shift-tab backward to discover each page's content. This is the user's
    very first impression of QUILL.
    Fix: on page change, announce "Step N of 9: <page title>" and focus the
    page's first meaningful control (heading or first input), not the nav
    button.

### Notable positives (keep these)

- context_help.py is exemplary SR design: help dialog title carries the
  control context so the announcement is immediate, body text receives focus.
- GitHub/SSH announce flows in the MainFrame mixins (download start, commit
  done, errors) are done right — announce + status together.
- Dialog contract infrastructure (apply_modal_ids, focus_primary_control,
  inventory gate, focus routing guard) is genuinely strong and tested.

### Medium

45. UX: `run_setup_wizard` calls ShowModal directly — first-run path skips the
    z-order gate that exists precisely to stop dialogs opening behind the
    frame (a bug class this project has already fixed once).
46. A11Y: console window (QDC) gaps consolidated from Pass 1: frame-level key
    handling (10), focus-return-to-editor (11), history announcement (17),
    copy-transcript announcement (7).
47. ENH (delight): QDC could announce "Console ready" when the TS worker
    finishes starting (currently only the silent status label changes).

## Pass 7: stability + platform/windows — COMPLETE

What's good: storage.py is exemplary (atomic replace with Windows AV-lock retry,
path-escape guard); task_manager has clean cancellation semantics; redaction
and safe_subprocess are well-documented and tested; prism_bridge's
don't-talk-over-the-screen-reader logic (auto mode checks SR presence, macOS
routes to VoiceOver) is exactly right; all 8 bundled quillins pass strict lint;
io transports all use verified_ssl_context.

### Critical

51. BUG (no crash feedback, app-wide): there is NO sys.excepthook anywhere in
    the codebase. An unhandled exception on the UI thread or at startup
    (before wx is up) in the windowed build means: no dialog, no speech, no
    crash bundle — QUILL just vanishes. For a blind user, double-clicking the
    app and having nothing happen with zero feedback is the worst possible
    failure mode. crash_report.py exists but is only reachable through the
    Help menu.
    Fix: install sys.excepthook (and wx.App.OnExceptionInMainLoop) that
    (a) writes a crash bundle, (b) shows ctypes.windll.user32.MessageBoxW —
    which Narrator/NVDA read even with no wx — telling the user where the
    crash report is.

52. A11Y/PERF BUG: prism_bridge.announce() in TTS-fallback mode calls
    pyttsx3 engine.runAndWait() — this BLOCKS the calling thread (the UI
    thread) until speech finishes. Every announcement freezes QUILL for the
    duration of the spoken text for non-screen-reader users (low vision users
    relying on self-voicing). A two-sentence announcement = ~5s frozen UI.
    Fix: dedicated speech worker thread with a queue.

### High

53. SEC: redaction misses modern API key formats. _TOKEN_RE only matches
    hex-ish strings ([A-Fa-f0-9_-]{32,}) — GitHub PATs (ghp_..., github_pat_),
    OpenAI keys (sk-proj-...), and AWS keys (AKIA...) contain letters outside
    a-f, so a BARE key on a log line (no name= prefix) passes through
    redaction into the crash bundle. Add prefix patterns: sk-, ghp_, gho_,
    github_pat_, AKIA, xox[bp]-, and a generic [A-Za-z0-9_-]{36,} catch.

54. BUG: _screen_reader_active() caches forever. Start NVDA after QUILL
    launches and QUILL keeps self-voicing on top of it until restart (the
    exact double-talk the cache comment says it prevents). Re-probe on a
    timer or invalidate on announce-error/backend reconfigure.

55. UX: safe_subprocess (and ts_console, finding 8) launch without
    CREATE_NO_WINDOW on Windows — every Piper/Pandoc/Tesseract call can flash
    a console window in the packaged GUI build. One creationflags addition in
    run_subprocess_safely fixes most call sites at once.

### Medium

56. ENH: write_json_atomic doesn't fsync before os.replace — a power cut in
    the window can leave a zero-byte settings file. One os.fsync(fh.fileno())
    closes it; cheap insurance for the settings/recovery files.
57. UX: _launch_configuration silently drops nonexistent CLI paths — open a
    just-deleted file from Explorer's recent list and QUILL starts empty with
    no explanation. Announce "Could not open <name>: file not found."
